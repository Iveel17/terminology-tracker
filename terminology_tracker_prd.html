Product Requirements Document

Terminology Tracker

A personal glossary tool for managing course terminology across 8 subjects

Version  1.0 Status  Draft Owner  Personal / Solo Deploy  Vercel Storage  localStorage

Overview

Terminology Tracker is a single-user personal web app for capturing and organizing course terminology across multiple subjects. Each week, new terms pile up — this tool gives them a home with definitions, examples, subject tags, week labels, and optional freehand sketch areas to visualize meaning.

Core premise: No backend, no auth, no database. Everything lives in the browser via localStorage. Deploy once to Vercel, use forever for free.

Problem Statement

Across 8 concurrent courses, new terminology accumulates weekly at a rate that makes it impossible to track via scattered notes. Current pain points:

  • Terms are siloed per course with no unified view
  • No quick way to search across all subjects
  • Text definitions alone are insufficient — visual sketches help reinforce meaning
  • Week-by-week context is lost over time

Data Model

Each terminology entry is stored as a JSON object. Sketch data is stored as a base64 canvas blob per entry. All entries live in a single localStorage key as an array.

// localStorage key: "terminology_entries"

{
  id: string,           // uuid
  term: string,         // terminology name
  subject: string,      // course / subject name
  week: number,         // week number
  definition: string,  // plain text definition
  example: string,     // usage example
  sketchData: string | null, // base64 canvas image
  sketchOpen: boolean,  // draw panel visible
  createdAt: string,   // ISO timestamp
}

Features

F1 — Terminology Entry Row

Core

Add / Edit / Delete a Term

  • Single-row form with fields: Term, Subject, Week #, Definition, Example
  • Inline editing — click any field to edit in place, auto-save on blur
  • Delete button per row with a simple confirmation
  • "+ Add Term" button at the top always visible
  • New entries appear at the top of the list

F2 — Toggleable Draw Section

Core

Per-row freehand sketch canvas

  • Each row has a "Draw" toggle button — clicking it expands a canvas panel beneath that row
  • Canvas is a freehand drawing surface (mouse + touch support)
  • Tool options: pen color picker, eraser, clear canvas
  • Sketch is saved to localStorage on every stroke (debounced)
  • If a sketch exists, the toggle button shows a small filled indicator so you know it has content even when collapsed
  • Collapsed by default; open state is preserved per session

F3 — Filter & Search Bar

Core

Find terms fast

  • Search: live text filter across term name, definition, and example
  • Filter by Subject: dropdown populated dynamically from existing subjects
  • Filter by Week: dropdown populated dynamically from existing week numbers
  • Filters are combinable (e.g. subject = "Stats" AND week = 3)
  • Active filter count badge shown so you know when filters are applied
  • One-click "Clear all filters" button

F4 — Export / Import

Nice to have

Backup and restore

  • Export all entries as a .json file (includes sketch data)
  • Import from a previously exported .json file — merges or replaces
  • Useful before clearing browser data or switching devices

Technical Constraints

Stack

Recommended

  • React (Vite or Next.js)
  • Tailwind CSS
  • HTML5 Canvas API for drawing
  • No database, no auth
Deployment

Vercel

  • Static site deploy (free tier)
  • No backend / serverless needed
  • All data stays in browser
localStorage limit: ~5MB per origin. Sketch data (base64 canvas) is the largest cost. Recommend capping canvas at 600×300px and compressing to JPEG quality 0.7 on save. Export/Import feature is the safety valve if storage fills up.

Out of Scope (v1)

  • User accounts / authentication
  • Cloud sync or database
  • Flashcard / quiz mode
  • Sharing / collaboration
  • Rich text editor (Markdown, bold, etc.)
  • Mobile-native app
